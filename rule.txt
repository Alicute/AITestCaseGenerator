# 测试项目框架创建公约

为了确保生成测试用例的结构一致性和可读性，请遵循以下约定：

## 1. 项目名称
- 所有测试用例的文档应以一级标题 `#` 开头，后面跟随项目名称。
  
  示例：
  ```
  # 项目名称
  ```

## 2. 一级模块
- 每个一级模块应使用二级标题 `##` 表示，后面跟随模块名称。
  
  示例：
  ```
  ## 一级模块名称
  ```

## 3. 三级模块
- 每个三级模块应使用三级标题 `###` 表示，后面跟随子模块名称。
  
  示例：
  ```
  ### 三级模块名称
  ```

## 4. 功能点
- 在每个三级模块下，使用无序列表 `-` 列出该模块的功能点。每个功能点应简洁明了，描述该功能的核心内容。
  
  示例：
  ```
  - 功能点1
  - 功能点2
  ```

## 5. 示例结构
以下是一个完整的示例结构，供参考：

```
# 项目名称

## 一级模块名称

### 三级模块名称
- 功能点1
- 功能点2

### 另一个三级模块名称
- 功能点3
- 功能点4
```

请大家在创建测试用例模块大纲时遵循以上公约，以确保文档的统一性和可维护性。谢谢大家的配合！
