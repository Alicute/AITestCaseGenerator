# 测试管理功能开发方案

## 背景

当前系统已有：项目 / 模块 / 测试用例 / AI生成，但缺少：
- 测试用例分层标记（冒烟/回归/全量）
- 测试计划与版本关联
- 用例执行记录与结果追踪
- 测试报告

本文档描述两个阶段的开发方案，供审核确认后执行。

---

## 阶段一：用例标签（labels）

### 目标
给测试用例和模块加多选标签，支持按标签筛选，实现冒烟/回归/全量用例集的快速圈定。支持三层标签继承机制，避免500+用例逐条手动标注。

### 标签定义

| 标签值 | 说明 |
|--------|------|
| `smoke` | 冒烟用例，每次发布前必跑 |
| `regression` | 回归用例，版本迭代时跑 |
| `full` | 全量用例，正式测试时跑 |

一条用例可同时拥有多个标签，例如：`["smoke", "regression"]`

---

### 标签继承机制（解决批量标注问题）

用例的最终有效标签按以下优先级计算：

```
用例自身 labels（显式设置，最高优先级）
  ↓ 若为 null（未设置）
所属模块的 labels（继承模块标签）
  ↓ 若模块也未设置
按 priority 字段自动推断（兜底）
  P0 → [smoke, regression, full]
  P1 → [regression, full]
  P2/P3/P4 → [full]
```

**排除场景**：用例显式设置 `labels: []`（空数组）表示不参与任何测试计划，优先级高于模块继承和自动推断。

**type字段排除**：创建测试计划时支持排除指定 `type`，如排除"性能测试"、"兼容性测试"，避免这类用例混入冒烟计划。

### 后端改动

#### 1. 数据库迁移
```sql
-- 用例表加 labels 字段
ALTER TABLE TestCases ADD COLUMN labels JSON DEFAULT NULL COMMENT '测试标签，null=继承模块，[]=不参与任何计划';

-- 模块表加 labels 字段
ALTER TABLE Modules ADD COLUMN labels JSON DEFAULT NULL COMMENT '模块级默认标签，用例未设置时继承此值';
```

#### 2. 模型修改

`backend/models/TestCase.js` 新增：
```js
labels: {
  type: DataTypes.JSON,
  allowNull: true,
  defaultValue: null  // null=继承模块标签
}
```

`backend/models/Module.js` 新增：
```js
labels: {
  type: DataTypes.JSON,
  allowNull: true,
  defaultValue: null
}
```

#### 3. 接口修改（`backend/controllers/testCaseController.js`）

**创建/更新**：透传 `labels` 字段。

**查询筛选**：`getTestCases` 支持 `?labels=smoke` 参数，查询时计算有效标签（含继承逻辑）：
```js
if (labels) {
  where[Op.and] = sequelize.literal(`JSON_CONTAINS(labels, '"${labels}"')`)
}
```

#### 4. 新增批量打标签接口

供AI或人工批量操作，无需逐条编辑：

```
POST /api/v1/testcases/batch-labels
Body: {
  "ids": [1, 2, 3],          // 指定用例ID（与moduleId/projectId三选一）
  "moduleId": 5,              // 按模块批量
  "projectId": 1,             // 按项目批量
  "labels": ["smoke", "regression"],
  "mode": "merge"             // merge=追加已有标签 / replace=完全覆盖
}

POST /api/v1/modules/:id/labels
Body: {
  "labels": ["regression", "full"]   // 设置模块级默认标签
}
```

## 阶段二：测试计划 + 执行记录

### 目标
每次测试活动对应一个测试计划，关联版本号，记录每条用例的执行结果，最终生成测试报告。

### 工作流

```
创建测试计划（版本+类型）
    ↓
从项目用例中按标签/模块勾选用例
    ↓
逐条执行，标记 通过/失败/跳过
    ↓
失败时填写禅道Bug ID
    ↓
计划完成，查看测试报告（通过率、失败列表）
```

---

### 数据模型

#### TestPlan（测试计划表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| name | VARCHAR(200) | 计划名称，如"v1.2.0冒烟测试" |
| version | VARCHAR(50) | 版本号，如"v1.2.0" |
| projectId | INT | 所属项目 |
| type | ENUM | 类型：smoke / regression / full |
| status | ENUM | 状态：draft / active / completed |
| createdBy | INT | 创建人 |
| createdAt | DATETIME | 创建时间 |

#### TestExecution（执行记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| planId | INT | 关联测试计划 |
| testCaseId | INT | 关联测试用例（快照存title） |
| testCaseTitle | VARCHAR(200) | 用例标题快照（防止用例被改后记录失真） |
| result | ENUM | 结果：pending / pass / fail / skip |
| defectId | VARCHAR(100) | 禅道Bug ID（失败时填写） |
| defectUrl | VARCHAR(500) | 禅道Bug链接（自动拼接） |
| remark | TEXT | 备注 |
| executedBy | INT | 执行人 |
| executedAt | DATETIME | 执行时间 |

4. **报告页面**：通过率环形图、失败用例列表（含禅道链接）
